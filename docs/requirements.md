# 飞书组织数据后台需求

## 目标

把现有一次性脚本升级为单机版 Web 后台，用来管理多个飞书公司主体，定时同步组织架构和员工信息，并保留员工从首次出现到离职为止的历史变更记录。

## 已确认约束

- 技术栈：Next.js Web 应用。
- 数据库：SQLite，本地文件持久化。
- 部署：Docker Compose。
- 访问控制：所有后台页面需要输入一个访问密码。
- 飞书凭证：每个公司主体维护自己的 `APP_ID` 和 `APP_SECRET`，`APP_SECRET` 明文入库。
- 配置：支持 `.env` 覆盖运行配置。
- 形态：单机版，不引入独立账号体系、消息队列或外部数据库。

## 功能需求

1. 公司主体管理
   - 后台可新增、编辑公司主体。
   - 每个主体保存名称、`APP_ID`、`APP_SECRET`、启用状态、同步配置。

2. 数据同步
   - 支持手动触发某个公司同步。
   - 支持按公司配置定时同步。
   - 同步部门树、用户当前信息、上下级关系和原始扩展字段。
   - 记录每次同步任务的状态、开始时间、结束时间、统计和错误信息。

3. 用户历史
   - 保存用户当前快照。
   - 每次同步与上一次快照比较，记录新增、字段变更、恢复在职、离职事件。
   - 用户在同步结果中消失时，标记为离职并记录离职时间。

4. 用户查询
   - 以公司为维度查询员工。
   - 默认展示在职用户。
   - 可切换查看所有用户。
   - 展示用户当前信息：姓名、邮箱、岗位、部门、上级、状态、最近同步时间。
   - 可查看单个用户的历史变更。

5. 部门树
   - 以公司为维度展示部门树。
   - 点击部门查看直属员工。
   - 展示员工与上级关系。

## 非目标

- 暂不做多用户登录、角色权限和操作审计。
- 暂不做 APP_SECRET 加密或密钥托管。
- 暂不做多实例高可用部署。
- 暂不做复杂 cron 表达式编辑器，第一版采用固定间隔分钟数。
